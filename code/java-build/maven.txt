بیلد برنامه های Maven‪-‬based در خط فرمان(CLI)‪:‬

۱- استفاده از برنامه نصبی mvn و اجرای دصتور ‪$‬mvn در پوشه پروژه های Maven
۲- استفاده از اسکریپت ‪./‬mvnw در خط فرمان

نکته ها: 

‪-‬ استفاده از روش mvnw حتی اگر روی سیستم مربوطه نسخه ای از Maven نصب نشده باشد باعث دانلود اتوماتیک یک نسخه از Maven و استفاده از آن در فرایند بیلد بصورت اتوماتیک می شود‪.‬

- در صورتی که در سیستم عامل ویندوز دستور اجرای shell script را می خواهیم اجرا کنیم از نسخه ‪.‬bat آن پروژه استفاده می کنیم.

- در صورتی در سیستم عامل Unix‪-‬based خطای اجرای فرمان ‪./‬mvnw بصورت Permission Deined دریافت می شود باید قبل از اجرای اسکریپت با دستور ‪$‬ chmod ‪+‬x mvnw اجازه اجرای فایل اسکریپت بیلد پروژه را روی سیستم مربوطه داد.

- در صورتی که در فرایند بیلد خطای عدم دسترسی به فایل های خاصی از POM های Dependency های پروژه روی سرورها صادر می شود می توان با اجرای دستور ‪$‬ ‪./‬mvnw ‪-‬X متوجه شد که دقیقا روی کدام کتابخانه و با خطای کدام سرور مواجه هستیم. اغلب این موضوع به دلیل تحریم IP های ایران توسط Repository Manager های مانند Google مثلا برای Dependency های پروژه های وابسته به گوگل مثلا آدرس های developer‪.‬google‪.‬com می باشد که در چنین شرایطی با باز کردن VPN های غیر ایرانی می توان موضوع را مرتفع کرد یا در ساختارهای سازمانی به دنبال راه اندازی Repository های داخلی شرکت بود. ابزارهای مانند JFrog که می توانند سرورهای داخلی Repository باشند.

۳- بعد از بیلد پروژه با مراجعه به پوشه target ایجاد شده در کنار فایل های POM‪.‬xml می توان در صورتی که خروجی پروژه بصورت یک executable JAR باشد آن را با دستور ‪$‬ java ‪-‬jar ‪[‬JAR‪_‬FILE‪_‬NAME‪].‬jar اجرا نمود.


در خصوص استفاده از دستور mvn به جای mvnw دقیقا فرایند مشابه است به جز اینکه باید قبل از اجرای این دستور از نسخه نصب شده mvn در لیست Environment Variable ها اطمینان حاصل کرد. مثلا با دستور mvn ‪-‬v می توانیم اطمینان حاصل کنیم که روی سیستم عامل ما یک نسخه از MVN 2‪.‬x|3‪.‬x قابل دسترس باشد. در غیر این صورت باید یک نسخه از آن را دانلود و در مسیری از سیستم unzip و با دستور export در لیست Environment Variable ها قرار داد.
